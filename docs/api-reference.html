🏭 LendefiMarketFactory

Factory contract for deploying new isolated lending markets. Each market is deployed with its own set of contracts for complete market isolation.

Core Functions

external createMarket

Creates a new isolated lending market for a specific base asset.

function createMarket(address baseAsset, string memory name, string memory symbol) external returns (address marketCore, address marketVault, address marketAssets)
Parameters
Parameter Type Description
baseAsset address Address of the ERC20 token to use as the market's base asset
name string Name for the market vault tokens (e.g., "Lendefi USDC Market")
symbol string Symbol for the market vault tokens (e.g., "lenUSDC")
Returns
Name Type Description
marketCore address Address of the deployed LendefiCore contract
marketVault address Address of the deployed LendefiMarketVault contract
marketAssets address Address of the deployed LendefiAssets contract
Note: Only addresses with MARKET_CREATOR_ROLE can create new markets. Each base asset can only have one market.

view getMarketsByAsset

Retrieves the market contracts for a specific base asset.

function getMarketsByAsset(address baseAsset) external view returns (address core, address vault, address assets)
Parameters
Parameter Type Description
baseAsset address Address of the base asset token

view getAllMarkets

Returns arrays of all deployed market contracts.

function getAllMarkets() external view returns (address[] memory cores, address[] memory vaults, address[] memory assets)
Gas Consideration: This function can be expensive for large numbers of markets. Consider using pagination for frontends.

🎯 LendefiCore

Core lending contract that manages positions, collateral, and borrowing logic. Users interact primarily with this contract for position management.

Position Management

external createPosition

Creates a new borrowing position with an isolated vault contract.

function createPosition(address collateralAsset, bool crossBorrow) external returns (uint256 positionId)
Parameters
Parameter Type Description
collateralAsset address Address of the initial collateral asset
crossBorrow bool Whether to enable cross-collateral borrowing (false for isolated)
Returns
Name Type Description
positionId uint256 Unique identifier for the created position 
// Example: Create a cross-collateral position with WETH
uint256 positionId = marketCore.createPosition(
    0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2, // WETH address
    true  // Enable cross-collateral borrowing
);

external supplyCollateral

Adds collateral to an existing position.

function supplyCollateral(address asset, uint256 amount, uint256 positionId) external
Parameters
Parameter Type Description
asset address Address of the collateral asset to supply
amount uint256 Amount of collateral to supply (in asset's decimals)
positionId uint256 ID of the position to add collateral to
Approval Required: You must approve the position vault contract to transfer your tokens before calling this function.

external borrow

Borrows base asset against collateral in a position.

function borrow(uint256 amount, uint256 positionId, uint256 maxSlippage) external
Parameters
Parameter Type Description
amount uint256 Amount of base asset to borrow
positionId uint256 ID of the position to borrow against
maxSlippage uint256 Maximum allowed slippage in basis points (e.g., 100 = 1%)

external repay

Repays borrowed base asset for a position.

function repay(uint256 amount, uint256 positionId) external
Tip: Pass type(uint256).max as amount to repay the full debt including accrued interest.

external withdrawCollateral

Withdraws collateral from a position (subject to health factor constraints).

function withdrawCollateral(address asset, uint256 amount, uint256 positionId) external

external closePosition

Closes a position by repaying all debt and withdrawing all collateral.

function closePosition(uint256 positionId) external
Prerequisites: Position must have zero debt and zero collateral to be closed.

View Functions

view getPositionInfo

Returns complete information about a position.

function getPositionInfo(uint256 positionId) external view returns (PositionInfo memory)
Returns PositionInfo Struct
Field Type Description
owner address Address of the position owner
vault address Address of the position's vault contract
debt uint256 Current debt amount including accrued interest
collateralValue uint256 Total collateral value in USD (8 decimals)
healthFactor uint256 Position health factor (18 decimals, <1e18 = liquidatable)
status PositionStatus Current status (ACTIVE, LIQUIDATED, CLOSED)

view calculateBorrowRate

Calculates the current borrow rate for a position based on utilization and collateral tier.

function calculateBorrowRate(uint256 positionId) external view returns (uint256 borrowRate)
Rate Formula: Base Rate + (Utilization Rate × Multiplier) + (Tier Jump Rate)

Liquidation Functions

external liquidatePosition

Liquidates an unhealthy position (health factor < 1.0).

function liquidatePosition(uint256 positionId, address collateralAsset, uint256 maxDebtToCover) external
Parameters
Parameter Type Description
positionId uint256 ID of the position to liquidate
collateralAsset address Collateral asset to receive as liquidation bonus
maxDebtToCover uint256 Maximum debt amount to cover (0 = liquidate all)
Liquidator Requirements: Must hold at least 20,000 governance tokens to perform liquidations.

🏦 LendefiMarketVault

ERC-4626 compliant vault for liquidity providers. Handles deposits, withdrawals, and flash loans for the market.

ERC-4626 Vault Functions

external deposit

Deposits base assets into the vault and mints vault shares.

function deposit(uint256 assets, address receiver) external returns (uint256 shares)
Parameters
Parameter Type Description
assets uint256 Amount of base assets to deposit
receiver address Address to receive the minted vault shares
MEV Protection: Same-block operations are prevented to protect against MEV attacks.

external withdraw

Burns vault shares to withdraw base assets.

function withdraw(uint256 assets, address receiver, address owner) external returns (uint256 shares)
Liquidity Check: Withdrawal may fail if insufficient liquidity is available in the vault.

view totalAssets

Returns the total amount of underlying assets managed by the vault.

function totalAssets() external view returns (uint256)

view convertToShares

Converts asset amount to vault shares using current exchange rate.

function convertToShares(uint256 assets) external view returns (uint256 shares)

view convertToAssets

Converts vault shares to asset amount using current exchange rate.

function convertToAssets(uint256 shares) external view returns (uint256 assets)

Flash Loan Functions

external flashLoan

Provides uncollateralized flash loans with configurable fees.

function flashLoan(address receiver, uint256 amount, bytes calldata params) external
Parameters
Parameter Type Description
receiver address Contract that will receive the flash loan
amount uint256 Amount of base asset to flash loan
params bytes Arbitrary data passed to the receiver contract 
// Example: Flash loan 100,000 USDC
bytes memory data = abi.encode(targetDex, minProfit);
vault.flashLoan(
    address(this),  // receiver contract
    100000e6,       // 100,000 USDC
    data           // encoded parameters
);
Fee: Default flash loan fee is 9 basis points (0.09%). Receiver must implement IFlashLoanReceiver interface.

Interest Rate Functions

view getSupplyRate

Calculates the current annual supply rate for liquidity providers.

function getSupplyRate() external view returns (uint256 supplyRate)
Rate Formula: Supply Rate = Borrow Rate × Utilization × (1 - Protocol Fee)

view getUtilizationRate

Returns the current utilization rate of the vault.

function getUtilizationRate() external view returns (uint256 utilizationRate)
Calculation: Utilization = Total Borrowed / (Total Supplied + Total Borrowed)

⚡ IFlashLoanReceiver Interface

Interface that must be implemented by contracts receiving flash loans from Lendefi Markets.

external executeOperation

Callback function executed during flash loan. Must return true for successful execution.

function executeOperation(address asset, uint256 amount, uint256 fee, address initiator, bytes calldata params) external returns (bool)
Parameters
Parameter Type Description
asset address Address of the flash loaned asset
amount uint256 Amount of assets flash loaned
fee uint256 Fee amount that must be paid
initiator address Address that initiated the flash loan
params bytes Arbitrary data passed from flash loan call 
contract MyFlashLoanReceiver is IFlashLoanReceiver {
    function executeOperation(
        address asset,
        uint256 amount,
        uint256 fee,
        address initiator,
        bytes calldata params
    ) external override returns (bool) {
        // Verify the caller is authorized
        require(msg.sender == vaultAddress, "Unauthorized");
        
        // Your flash loan logic here
        // ...
        
        // Ensure we have amount + fee to repay
        uint256 totalRepayment = amount + fee;
        require(
            IERC20(asset).balanceOf(address(this)) >= totalRepayment,
            "Insufficient funds for repayment"
        );
        
        return true; // Must return true for success
    }
}
Important: The vault checks your contract's balance after this function returns. Ensure you have amount + fee available.

Individual position vault contracts deployed via minimal proxy pattern. Each position gets its own isolated vault for complete asset segregation.

Core Functions

external initialize

Initializes a new position vault with owner and core contract addresses.

function initialize(address owner, address core, bool crossBorrow) external
Parameters
Parameter Type Description
owner address Address of the position owner
core address Address of the LendefiCore contract
crossBorrow bool Whether cross-collateral borrowing is enabled
Note: This function can only be called once during vault deployment.

external deposit

Deposits collateral assets into the position vault.

Access Control: Only the position owner or authorized core contract can call this function.

📊 Types & Enums

CollateralTier Enum

Collateral Risk Tiers

Classification system for collateral assets based on risk and borrowing capabilities.

Tier Value Description Typical Assets
STABLE 0 Stable assets with lowest risk USDC, USDT, USD1
CROSS_A 1 High-quality cross-collateral assets WETH, WBTC
CROSS_B 2 Medium-quality cross-collateral assets Major DeFi tokens
ISOLATED 3 Isolated collateral with debt caps Long-tail assets

PositionStatus Enum

Position Lifecycle States

Status Value Description
INACTIVE 0 Position created but no collateral supplied
ACTIVE 1 Position has collateral and may have debt
LIQUIDATED 2 Position was liquidated due to low health factor
CLOSED 3 Position closed by owner with zero debt and collateral

OracleType Enum

Supported Oracle Types

Type Value Description
CHAINLINK 0 Chainlink price feeds (primary oracle)
UNISWAP_V3 1 Uniswap V3 TWAP oracle (fallback)
